## 1. Executive Summary

**KooD3plot Analysis Platform**은 LS-DYNA 유한요소 해석 결과를 **완전 자동**으로 분석하고, 엔지니어링 리포트를 생성하는 통합 플랫폼이다. 두 개의 핵심 모듈로 구성된다:

1. **Single Analyzer** — 단일 시뮬레이션 심층 분석 (응력, 변형률, 운동, 에너지, 접촉, 렌더링)
2. **KooReport** — 다각도 낙하 시험의 전구면(全球面) 분석 (Mollweide 투영, 히트맵, 충격 펄스, 균열 정지 평가)

### 기존 워크플로우의 한계

| 단계 | 기존 수동 작업 | 소요 시간 |
|------|--------------|----------|
| 파일 탐색 | 디렉토리별 d3plot/glstat/binout 수동 확인 | 5~10분/케이스 |
| 응력 확인 | LSPrePost GUI 실행 → 부품 선택 → fringe 설정 → 값 확인 반복 | 15~30분/부품 |
| 에너지 밸런스 | glstat 텍스트 파일 수동 파싱 + 엑셀 차트 | 20~40분 |
| 접촉력 확인 | binout 파싱 도구 별도 실행 | 10~20분 |
| 렌더링 | GUI에서 단면 설정 → 녹화 → 축 변경 반복 | 5~10분/영상 |
| 리포트 | 스크린샷 + 엑셀 + PPT 수작업 | 1~2시간 |
| **26각도 분석** | 위 전체 × 26 반복 | **2~4일** |


## 3. 핵심 기술 원리

### 3.1 D3plot 바이너리 포맷 파싱

LS-DYNA의 d3plot은 **비공개 바이너리 포맷**으로, 공식 문서(LS-DYNA User's Manual Appendix)를 기반으로 역공학적으로 파싱한다.

#### 3.1.1 포맷 자동 감지

d3plot 파일은 **단정밀도(4바이트)/배정밀도(8바이트)** 및 **리틀/빅 엔디안** 조합이 가능하다. KooD3plot은 4가지 조합을 모두 시도하여 자동 감지한다:

```
시도 1: 단정밀도 + 리틀 엔디안 → 워드 14에서 버전 읽기
시도 2: 단정밀도 + 빅 엔디안   → 바이트 스왑 후 버전 읽기
시도 3: 배정밀도 + 리틀 엔디안 → 워드 14에서 버전 읽기
시도 4: 배정밀도 + 빅 엔디안   → 바이트 스왑 후 버전 읽기
```

검증: 읽은 값이 LS-DYNA 버전 범위(900.0~2000.0) 내에 있으면 해당 포맷으로 확정.

#### 3.1.2 제어 워드 (Control Data)

d3plot의 **첫 64+EXTRA 워드**가 전체 데이터 레이아웃을 결정한다:

| 워드 | 이름 | 의미 |
|------|------|------|
| 15 | NDIM | 차원 (3D=3, 특수=4/5/7) |
| 16 | NUMNP | 절점 수 |
| 19~22 | IT, IU, IV, IA | 온도/변위/속도/가속도 출력 플래그 |
| 23 | NEL8 | 8절점 솔리드 요소 수 (부호 비트에 정보 인코딩) |
| 27 | NV3D | 솔리드 요소당 변수 수 |
| 31 | NEL4 | 4절점 쉘 요소 수 |
| 33 | NV2D | 쉘 요소당 변수 수 |
| 36 | MAXINT | 적분점 수 (부호 비트에 MDLOPT 인코딩) |
| 51 | NMMAT | 재료/부품 총 수 |

이 제어 워드들로부터 **파생 값**을 계산하여 데이터 접근 오프셋을 결정한다:

```
NND = ((IT+N) + effective_ndim×(IU+IV+IA)) × NUMNP    # 절점 데이터 크기
ENN = |NEL8|×NV3D + NELT×NV3DT + NEL2×NV1D + NEL4×NV2D   # 요소 데이터 크기
상태 블록 크기 = 1(시간) + NGLBV + NND + ENN + DELNN
```

#### 3.1.3 Family 파일 구조

대규모 시뮬레이션은 단일 d3plot 파일을 넘어 **family 파일**로 분할된다:

```
d3plot      ← 베이스 파일 (제어 데이터 + 지오메트리 + 초기 상태)
d3plot01    ← 상태 데이터만 (지오메트리 없음)
d3plot02
...
d3plot52    ← 52개 파일 = 약 3GB 데이터
```

KooD3plot은 이 family 파일들을 **병렬로 독립 읽기**한다. 각 파일은 지오메트리 없이 상태 블록만 포함하므로, 파일별 독립 파싱이 가능하다.

#### 3.1.4 병렬 읽기 알고리즘

```
Phase 1: 순차 (베이스 파일)
  ├── 포맷 감지 (4가지 조합)
  ├── 제어 데이터 파싱 (64+EXTRA 워드)
  └── 지오메트리 파싱 (절점 좌표 + 요소 연결)

Phase 2: 병렬 (std::async, Family 파일별 1스레드)
  ├── Thread 1: d3plot01 → 상태 5~20
  ├── Thread 2: d3plot02 → 상태 21~36
  ├── Thread 3: d3plot03 → 상태 37~52
  └── ...
  결과: 파일 인덱스 순서로 정렬 후 병합

성능: 52개 d3plot 파일 → 4스레드로 약 30초 완료
```

### 3.2 Von Mises 등가응력 계산

#### 3.2.1 응력 텐서 추출

d3plot의 솔리드 요소 데이터는 **NV3D 변수/요소** 구조로 저장된다. 처음 7개 변수가 응력 텐서 + 유효 소성 변형률:

```
워드 0: σxx (법선 응력 X)
워드 1: σyy (법선 응력 Y)
워드 2: σzz (법선 응력 Z)
워드 3: σxy (전단 응력 XY)
워드 4: σyz (전단 응력 YZ)
워드 5: σzx (전단 응력 ZX)
워드 6: εeff_plastic (유효 소성 변형률)
```

배열 접근: `base = element_index × NV3D`, `σxx = solid_data[base + 0]`

#### 3.2.2 Von Mises 공식

6성분 응력 텐서를 단일 스칼라 등가응력으로 변환한다:

```
σ_vm = √[½ × ((σxx-σyy)² + (σyy-σzz)² + (σzz-σxx)²) + 3 × (σxy² + σyz² + σzx²)]
```

C++ 구현:
```cpp
double d1 = sxx - syy;
double d2 = syy - szz;
double d3 = szz - sxx;
double vm = sqrt(0.5 * (d1*d1 + d2*d2 + d3*d3) + 3.0 * (sxy*sxy + syz*syz + szx*szx));
```

**물리적 의미**:
- σ_vm = 0: 무응력 상태
- σ_vm > 0: 편차(shape-changing) 응력 — 정수압(체적 변화) 성분 제외
- **좌표 회전 불변**: 동일한 응력 상태를 다른 좌표계에서 계산해도 같은 값
- **항복 기준**: σ_vm ≤ σ_yield 이면 탄성 영역 (Von Mises 항복 조건)

#### 3.2.3 주응력 및 고급 텐서 연산

KooD3plot은 Von Mises 외에도 다양한 텐서 기반 물리량을 계산한다:

| 물리량 | 공식 | 용도 |
|--------|------|------|
| 주응력 σ₁, σ₂, σ₃ | 응력 텐서의 고유값 (Lode angle 방법) | 파단 기준 평가 |
| 정수압 σ_m | (σxx + σyy + σzz) / 3 | 삼축 응력비 계산 |
| 삼축 응력비 | σ_m / σ_vm | 연성 파단 예측 |
| 최대 전단 응력 | (σ₁ - σ₃) / 2 | Tresca 항복 기준 |
| 표면 법선/전단 응력 | t = σ·n, σ_n = n·t, τ = √(\|t\|² - σ_n²) | 방향별 표면 분석 |

### 3.3 단일 패스 분석기 (Single-Pass Analyzer)

**핵심 아이디어**: 데이터를 **한 번만 순회**하며 복수 물리량을 동시에 계산한다.

```
상태 0 읽기 → [응력 max/min/avg] + [변형률 max/min] + [변위/속도/가속도] 동시 집계
상태 1 읽기 → 동 시 집계
...
상태 N 읽기 → 동시 집계
→ 모든 부품의 전체 시계열이 1회 순회로 완성
```

#### OpenMP 병렬화 전략

```cpp
#pragma omp parallel for schedule(dynamic, 1)
for (int64_t state = 0; state < num_states; ++state) {
    // 각 스레드는 독립적인 상태를 처리
    // 스레드-로컬 누적기로 경합(contention) 제거
    thread_local_stats[omp_get_thread_num()].accumulate(state_data);
}
// 병렬 구간 종료 후 병합
merge_all_thread_stats();
```

- **상태 수준 병렬**: 각 스레드가 서로 다른 타임스텝을 처리
- **요소 수준 순차**: 한 상태 내에서는 순차 처리 (중첩 병렬화 오버헤드 방지)
- **dynamic 스케줄링**: 상태별 작업량 불균형 대응

### 3.4 Mollweide 등적 투영 (KooReport)

전각도 낙하 시험 결과를 **구면 전체에 걸쳐 시각화**하기 위해 Mollweide 등적(equal-area) 투영을 구현한다.

#### 3.4.1 수학적 원리

Mollweide 투영은 **면적 보존** 투영법으로, 구면의 넓이가 투영면에서도 동일하게 유지된다.

투영 방정식:
```
x = (2√2 / π) × λ × cos(θ)
y = √2 × sin(θ)

여기서 θ는 보조각으로, 초월방정식을 풀어 구한다:
2θ + sin(2θ) = π × sin(φ)
```

φ = 위도(latitude), λ = 경도(longitude)

#### 3.4.2 Newton-Raphson 반복 해법

초월방정식 `f(θ) = 2θ + sin(2θ) - π·sin(φ) = 0`은 해석적 해가 없으므로, Newton-Raphson 반복법으로 수치 해를 구한다:

```javascript
function solveTheta(phi) {
    let theta = phi;  // 초기 추정
    for (let i = 0; i < 100; i++) {
        let f = 2*theta + Math.sin(2*theta) - Math.PI*Math.sin(phi);
        let fp = 2 + 2*Math.cos(2*theta);  // 도함수
        let delta = f / fp;
        theta -= 0.8 * delta;  // 감쇠 계수 0.8 (발산 방지)
        if (Math.abs(delta) < 1e-7) break;
    }
    return theta;
}
```

감쇠 계수 0.8을 적용하여 극점 근처에서의 오버슈팅을 방지한다.

#### 3.4.3 등고선 보간

측정된 이산 데이터점(각 낙하 각도의 결과값)을 연속적 컨투어로 변환한다:

- **역거리 가중(IDW) 보간**: 각 그리드점에서 주변 데이터점까지의 거리 역수로 가중 평균
- **방사 기저 함수(RBF)**: 더 부드러운 보간을 위한 선택적 적용
- 등고선 레벨: 자동 또는 수동 (10, 20, 50, 100 MPa 간격)

### 3.5 부품별 하이라이트 렌더링

#### 3.5.1 카메라 자동 센터링

LSPrePost 배치 모드에서 특정 부품에 최적 시점을 자동 설정한다:

```
Step 1: selectpart all off          → 모든 부품 숨기기
Step 2: selectpart on <PID>/0       → 타겟 부품만 표시
Step 3: fit                         → 화면에 맞추기 (bbox 기반)
Step 4: zout 1.500000               → 약간 줌아웃 (여백 확보)
Step 5: selectpart all on           → 모든 부품 복원 (카메라 유지!)
Step 6: 단면/컨투어 적용             → 렌더링 실행
```

**핵심 발견**: Step 5에서 모든 부품을 복원해도 **카메라 위치는 Step 3~4에서 설정한 상태로 유지**된다. 이것이 부품별 최적 시점의 핵심 메커니즘이다.

#### 3.5.2 축별 차별화 렌더링 전략

Mesa OpenGL(소프트웨어 렌더링)에서 `splane drawcut` 명령이 **Z축(수평) 단면에서만 정상 작동**하고, X/Y축(수직) 단면에서는 모델이 보이지 않는 버그가 있다. 이에 따라 축별로 다른 전략을 사용한다:

| 축 | 단면 | 컨투어 | 원리 |
|----|------|--------|------|
| **Z (수평)** | `drawcut` (실제 잘림) | `genselect` (타겟만 컬러) | drawcut이 Mesa에서 정상 작동 |
| **X/Y (수직)** | `splane` (표시만) | 글로벌 fringe + 부품별 범위 | drawcut 불가 → 전체 fringe 후 범위 조정 |

#### 3.5.3 부품별 Fringe 범위 자동 산출

X/Y축 렌더링 시, 전체 모델에 fringe를 적용하면 타겟 부품의 색상이 다른 부품의 극값에 의해 묻히는 문제가 발생한다. 이를 해결하기 위해:

```
1. ~5개 타임스텝 균등 샘플링 (0%, 25%, 50%, 75%, 100%)
2. 각 샘플 상태에서 타겟 부품 요소들의 Von Mises 계산
3. 전체 샘플에서 [0, max] 범위 산출
4. 이 범위로 LSPrePost fringe 컬러맵 고정: pfringe userdef <min> <max>
```

### 3.6 병렬 렌더링 아키텍처

각 LSPrePost 프로세스는 **독립적으로 d3plot를 로드**하므로, 여러 인스턴스를 동시 실행할 수 있다.

```
Phase 1: 태스크 준비 (순차, D3plotReader 접근)
  ├── Job 1 → Task 1, 2, 3 (overview X/Y/Z)
  ├── Job 2 → Task 4, 5, 6 (part_13 X/Y/Z)
  └── 총 72개 태스크 생성

Phase 2: 태스크 실행 (병렬, 독립 LSPrePost 프로세스)
  Thread 1: ┌─Task 1─┐ ┌─Task 4─┐ ┌─Task 7─┐ ...
  Thread 2: ┌─Task 2─┐ ┌─Task 5─┐ ┌─Task 8─┐ ...
  Thread 3: ┌─Task 3─┐ ┌─Task 6─┐ ┌─Task 9─┐ ...
  Thread 4: (대기 → Task 10 할당 → ...)

  태스크 큐: mutex로 보호된 공유 인덱스
  로깅: mutex로 보호된 callback 호출
```

구현: C++ `std::thread` + `std::mutex` 기반 워커 풀.

## 4. Single Analyzer — 상세 기능

### 4.1 자동 파일 탐지 및 Tier 시스템

시뮬레이션 디렉토리 구조를 자동 탐색하여 사용 가능한 파일 조합에 따라 분석 깊이를 결정한다.

| Tier | 필요 파일 | 분석 범위 | 자동 결정 로직 |
|------|----------|----------|--------------|
| **T0** | (오류 종료 감지) | **분석 불가** — 자동 스킵 | glstat 마지막 50줄에서 "Error termination" 패턴 감지 |
| **T1** | d3plot | 응력, 변형률, 변위, 속도, 가속도 | d3plot만 존재 |
| **T2** | d3plot + glstat | T1 + 에너지 밸런스, 질량 증가 감지 | glstat 파일 존재 |
| **T3** | d3plot + glstat + binout | T2 + 접촉력, 마찰 에너지 | binout(lasso) 파일 존재 |
| **T4** | d3plot + glstat + binout + rcforc/matsum | **완전 분석** | rcforc 또는 matsum 존재 |

**종료 상태 판별 알고리즘**:
1. glstat 파일의 마지막 ~6000바이트(50줄)를 읽기
2. 공백 제거 + 소문자 변환: `replace(' ','').lower()`
3. `"normaltermination"` 또는 `"errortermination"` 패턴 검색
4. glstat에 없으면 d3hsp 파일의 꼬리 부분 확인
5. 모두 없으면 `"unknown"` — 분석 계속 진행 (d3plot 존재 시)

### 4.2 분석 유형 6가지

#### (1) Von Mises 응력 분석 (`von_mises`)
- 전 부품 또는 지정 부품의 Von Mises 등가 응력 시계열
- 매 타임스텝별 **최대/최소/평균** + **최대 요소 ID** 추적
- 글로벌 피크 시점 및 위치 자동 식별
- 출력: CSV (`Time, Max_vonMises, Min_vonMises, Avg_vonMises, Max_Element_ID`)

#### (2) 유효 소성 변형률 (`eff_plastic_strain`)
- d3plot 워드 6에서 직접 추출 (별도 계산 불필요)
- 소성 변형 영역 추적 → 재료 파단 평가
- 변형률 > 0: 영구 변형 발생

#### (3) 부품 운동 분석 (`part_motion`)
- 무게중심(centroid) 기반 평균 변위/속도/가속도
- 최대 변위 노드 ID 추적 (극단 변형 위치 파악)
- **X/Y/Z 방향별 분리 출력** — 어느 축이 지배적인지 파악

물리량 목록:
- `avg_displacement`: 부품 전 절점의 평균 변위 벡터
- `avg_velocity`: 부품 전 절점의 평균 속도 크기
- `avg_acceleration`: 부품 전 절점의 평균 가속도 크기
- `max_displacement`: 부품 내 최대 변위 절점 (ID 포함)

#### (4) 표면 응력 분석 (`surface_stress`)
- 지정 방향(법선 벡터)과 허용 각도 이내의 표면 요소 자동 필터링
- 표면 법선 응력 σ_n = **n**·(**σ**·**n**) 및 전단 τ = √(|**t**|² - σ_n²) 계산
- 활용: 낙하 시 바닥면/측면/상면 응력 비교

#### (5) 표면 변형률 분석 (`surface_strain`)
- 표면 응력과 동일한 필터링 + 변형률 텐서 적용

#### (6) 통합 분석 (`comprehensive`)
- 복수 물리량을 단일 Job으로 지정
- 내부적으로 적절한 분석 유형들로 분할 실행

## 5. KooReport — 전각도 낙하 분석 상세

### 5.1 전각도 낙하 시험이란?

제품(전자기기, 배터리 팩 등)을 **모든 가능한 방향에서 낙하**시켜 방향 의존적 취약성을 평가하는 시험이다.

#### DOE (실험 계획법) 전략

| 전략 | 각도 수 | 구면 커버리지 | 특성 |
|------|---------|-------------|------|
| **Cuboid 26** | 26개 | ~12% | 면(6) + 모서리(12) + 꼭지점(8) — 산업 표준 |
| **Fibonacci 구면 샘플링** | N개 (자유) | 최대 100% | 피보나치 격자에 의한 준균일 분포 |

**피보나치 구면 샘플링**: N개 점을 구면에 **최대한 균일하게 분포**시키는 알고리즘. 최근접 이웃 간 거리의 변동 계수(CoV)가 ~3%로, 무작위 분포(~30%)보다 10배 균일하다.

#### 구면 커버리지 지표

```
이상 간격 = √(4π/N)  (N개 점의 이론적 최적 간격)
실제 간격 = 측정된 최근접 이웃 평균 거리
커버리지 = min(1.0, (이상 간격 / 실제 간격)²)
```

### 5.3 KooReport HTML 리포트 (12개 탭)

#### Tab 0: Overview & 리포트 가이드
- **통계 카드**: 완료 시뮬 수, DOE 전략, 각도 간격, 구면 커버리지, 낙하 높이
- **자동 Findings**: 색상 코딩 — CRITICAL(적), WARNING(황), INFO(청)
  - 예: "Part PKG 1의 피크 응력이 항복 응력 250 MPa를 초과합니다 (661 MPa, SF=0.38)"
- **리포트 해석 가이드**: Von Mises 정의, G-force 스케일, 변형률 해석, 안전계수 의미

#### Tab 1: Mollweide 투영 맵
- **등적 투영**: 구면의 전체 결과를 2:1 평면에 투영
- **배경 컨투어**: 역거리 가중(IDW) 보간으로 연속 색상 필드 생성
- **데이터점**: 각 낙하 각도의 위치 (색상 = 물리량 크기)
- **위도/경도 격자선**: 15° 간격

#### Tab 2: 시계열 (파형 비교)
- **포락선 모드**: 전 방향의 min/max/avg 시계열 오버레이
- **단일 방향 모드**: 선택한 각도들의 파형 비교
- **XYZ 성분 모드**: 가속도/변위의 축별 분해

자동 해상도 적응:
- ≤50 결과: 100 포인트/시리즈
- ≤200 결과: 30 포인트/시리즈
- \>200 결과: 20 포인트/시리즈

#### Tab 3: 부품 위험도 (Worst-Case)
- 부품별 **최악 각도**에서의 피크 응력/G/변형률 표
- 안전계수 색상 코딩

#### Tab 4: 히트맵 (부품 × 각도 매트릭스)
- 행 = 부품, 열 = 각도
- 셀 색상 = 물리량 크기 (Blue→Cyan→Yellow→Red)
- **패턴 인식**:
  - 가로 빨간 띠 = 취약 부품 (전 방향에서 높은 응력)
  - 세로 빨간 띠 = 위험 방향 (전 부품에 높은 응력)
  - 대각 집중 = 방향-부품 상관관계

#### Tab 5: 방향 민감도
- 방향별 심각도 랭킹
- 카테고리(면/모서리/꼭지점) 비교 박스플롯
- 대칭 검증: 전면 vs 후면, 좌측 vs 우측

#### Tab 6: 안전계수 분석
- 부품별 SF 바 차트 (적/황/녹)
- 피크 응력 vs G-force 산점도 (SF 색상)
- 설계 권고 자동 생성

#### Tab 7: 통계 분석
- 부품별 평균/표준편차/CoV
- 분포 히스토그램
- **피어슨 상관 매트릭스**: 응력↔G, 응력↔변형률, G↔변위 등

#### Tab 8: 충격 분석 (Impact Analysis)
- **충격 펄스 특성화**:
  - 펄스 폭 (ms): 가속도 10% 임계치 교차 기준
  - 피크 가속도 (MG)
  - 임펄스 (MG·ms): |가속도| × 시간 적분
  - **반사인 등가 펄스**: A·sin(πt/T) — MIL-STD-810G, IEC 60068-2-27 호환

- **반발 정량화**:
  - 바운스 수: 2차 피크(>15% of max) 카운트
  - 2차/1차 비율: 반발 계수(COR) 근사
  - 감쇠율 λ: 지수 감쇠 peak_n ∝ e^(-λt)

- **핫스팟 추적 (Cross-Part)**:
  - 요소 빈도 히스토그램: 어떤 요소가 가장 많은 각도에서 피크 응력을 경험하는가?
  - 빈도 >70%: **지오메트리 기반** (필렛, 노치, 접촉 에지)
  - 빈도 20~70%: **방향 의존적**
  - 빈도 <20%: **분산형** (단일 임계 요소 없음)

#### Tab 9: 부품 Deep Dive
- 단일 부품 선택 → 전 물리량 통합 대시보드
- 응력 포락선 + 항복 기준선
- 카테고리별 취약성 바 차트
- **자동 엔지니어링 내러티브** (한/영 이중 언어):
  > "PKG 1은 F3_Right 면 낙하에서 1250 MPa 최고응력을 겪으며 (평균 대비 40% 높음), 우측 구조적 취약성을 나타냅니다..."

#### Tab 10: 고급 분석 (7개 모듈)

1. **충격 펄스 특성화** — SRS(충격 응답 스펙트럼) 호환 반사인 등가 펄스
2. **핫스팟 추적** — 요소 빈도 히트맵, 응력 집중 계수
3. **에너지 흡수 분석** — 부품별 변형 에너지 W = ∫σ·dε, 에너지 분담률
4. **반발 정량화** — 다중 바운스 특성, 감쇠율, JEDEC 낙하 시험 기준 비교
5. **상호 상관 매트릭스** — 피어슨 r로 물리량 간 결합/독립 관계 파악
6. **균열 정지 감수성 (CAI 지수)** — 물리 기반 4요소 가중 점수:
   - 응력 지속 비율 (30%): 항복 초과 시간 비율
   - 하중 속도 점수 (20%): dσ/dt 초기 상승률
   - 응력 집중 점수 (25%): σ_max/σ_avg 비율
   - 에너지 소산 점수 (25%): 피크 후 응력 감쇠율
   - CAI > 0.6: 균열 정지 가능, 0.3~0.6: 경계, < 0.3: 불가
   - 참고 규격: BS 7910, ASTM E1820, API 579-1

7. **렌더 설정 내보내기** — 선택 각도의 YAML 생성

#### Tab 11: 렌더 설정 내보내기
- Mollweide 맵에서 각도 클릭 선택
- 빠른 선택: "Top 5 Stress", "Top 5 G-force"
- 렌더링 옵션 (축, 위치, 물리량, 해상도, FPS)
- YAML 미리보기 + 다운로드/복사

### 5.4 다국어 지원 (i18n)

## 7. AI 활용 상세

### 7.1 개발에서의 AI 역할

이 프로젝트는 **Claude AI (Anthropic)**를 핵심 개발 도구로 활용하여 구축되었다. AI는 단순 코드 생성을 넘어, **엔지니어링 의사결정과 문제 해결**에 깊이 관여했다.

#### 7.1.1 아키텍처 설계

| 의사결정 | AI 기여 |
|---------|---------|
| d3plot 파서를 외부 라이브러리 대신 자체 구현 | 외부 의존성 제거 전략 제안 + 바이너리 포맷 역공학 코드 작성 |
| 순환 의존성 방지 (libkood3plot ↔ libkood3plot_render) | `target_sources()` + `target_compile_definitions()` 패턴 제안 |
| 렌더링 코드를 실행 파일에 직접 컴파일 | CMake 구조 설계 및 `#ifdef KOOD3PLOT_HAS_RENDER` 가드 패턴 |
| 병렬 렌더링의 2-Phase 구조 | Phase 1(순차, reader 접근) + Phase 2(병렬, 독립 프로세스) 분리 설계 |

#### 7.1.2 알고리즘 구현

| 알고리즘 | AI 기여 |
|---------|---------|
| Von Mises 응력 계산 | 수학 공식 → C++ 최적화 코드 변환, 캐시 효율적 메모리 접근 패턴 |
| Mollweide 투영 | Newton-Raphson 감쇠 계수(0.8) 결정, 극점 수렴성 해결 |
| 역거리 가중 보간 | 구면 컨투어 생성을 위한 IDW 파라미터 튜닝 |
| CAI (균열 정지 감수성) 지수 | BS 7910/ASTM E1820 기반 4요소 물리 모델 설계 |
| 피보나치 구면 커버리지 | 이론적 최적 간격 √(4π/N) 대비 실측 간격 비교 공식 |

#### 7.1.3 문제 해결 (디버깅)

| 문제 | AI 해결 과정 |
|------|-----------|
| Mesa OpenGL에서 X/Y drawcut 시 모델 소실 | 축별 전략 분리(Z=drawcut, X/Y=splane+global fringe) 설계 |
| 부품별 카메라 센터링 실패 | selectpart → fit → zout 패턴 발견 및 검증 (기존 코드 대비 10줄→6줄) |
| Z축 drawcut + fit = 잘린 단면만 fit | fit를 drawcut 전에 실행하는 순서 최적화 |
| fringe/pfringe가 blank 상태 리셋 | blank+fringe 비호환 발견 → genselect 대체 전략 |
| X/Y축 타겟 부품 컨투어 묻힘 | 부품별 fringe 범위 자동 산출(5-state 샘플링) 구현 |
| 52파일 병렬 읽기 시 OOM (3GB) | `--ua-threads 4` 기본값 권장, 테스트 결과 기반 메모리 사용량 프로파일링 |

#### 7.1.4 리포트 시스템

| 기능 | AI 기여 |
|------|---------|
| 12탭 대시보드 UI | HTML/CSS/JS 전체 설계 + 코드 생성 (html_report.py: ~4,300줄) |
| 3D Globe 렌더링 | Canvas API 기반 직교 투영, WebGL 없이 구현 |
| 인터랙티브 히트맵 | HSL 색상 매핑, 호버/클릭 상호작용 |
| 자동 Findings | 규칙 기반 위험도 평가 로직 (CRITICAL/WARNING/INFO) |
| i18n 시스템 | 한/영 이중 언어 L() 함수 + I18N 사전 (~200키) |
