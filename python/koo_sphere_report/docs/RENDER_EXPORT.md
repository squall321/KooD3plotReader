# Render Export — 취약 각도 단면 영상 렌더링

## 개요

KooReport의 **Render Export** 탭(Tab 11)에서 Mollweide 맵의 취약 방향을 선택하고,
단면 축/위치/컨투어 물리량/출력 형식 등 렌더링 옵션을 지정하면
`unified_analyzer`가 처리할 수 있는 YAML 설정 파일을 자동 생성합니다.

**워크플로:**
```
Mollweide 맵에서 취약 각도 선택
    ↓
렌더링 옵션 지정 (축, 위치, 물리량, 형식)
    ↓
YAML 다운로드
    ↓
analyze_and_report --render-config render_config.yaml
    ↓
renders/ 폴더에 mp4/png/gif 파일 생성
```

---

## HTML 리포트 — Render Export 탭

### 접근
HTML 리포트의 마지막 탭 **"Render Export"** 를 클릭합니다.

### 패널 1: 각도 선택

| 요소 | 설명 |
|------|------|
| **참조 부품 드롭다운** | Quick Select 기준 부품. 이 부품의 응력/G 순위로 Top 5가 결정됩니다. |
| **Top 5 Stress** 버튼 | 선택된 부품의 최대 Von Mises 응력 상위 5개 각도 자동 선택 |
| **Top 5 G** 버튼 | G-force 상위 5개 각도 자동 선택 |
| **Clear** 버튼 | 선택 초기화 |
| **미니 Mollweide** | 클릭으로 각도 토글 (선택/해제). 선택된 점은 강조 표시됩니다. |
| **각도 칩** | 선택된 각도 이름 표시. `x` 클릭으로 개별 제거 |

### 패널 2: 렌더링 옵션

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| **Section Axis** | Z | 단면 절단 축 (X / Y / Z) |
| **Section Positions** | center | 절단 위치 (center, 25%, 50%, 75%, min, max 복수 선택) |
| **Custom Positions** | (비어 있음) | 추가 위치 직접 입력 (예: `30%, 60%`) |
| **Fringe Quantity** | Von Mises Stress | 컨투어 물리량 |
| **Part Filter** | `PKG*` | 부품 패턴 (glob) |
| **Output Format** | MP4 | 출력 형식 (MP4 / PNG / GIF) |
| **Resolution** | 1920×1080 | 출력 해상도 |
| **FPS** | 30 | 프레임 레이트 (MP4/GIF 전용) |

#### Fringe Quantity 목록
- Von Mises Stress
- Eff. Plastic Strain
- Displacement
- Velocity
- Acceleration
- Stress XX / YY / ZZ
- Principal Stress 1
- Max Shear

### 패널 3: YAML 미리보기 + 다운로드

- 실시간으로 생성되는 YAML을 syntax-highlighted로 미리보기
- **Download YAML** — `render_config_<프로젝트명>.yaml` 다운로드
- **Copy to Clipboard** — 클립보드 복사
- 하단에 실행 가이드 표시

## 생성되는 YAML 형식

```yaml
# Auto-generated by KooReport Render Export
# Test: Test_001_Full26_1Step
# Generated: 2026-02-08T15:30:00
version: "2.0"

performance:
  lsprepost_path: "/path/to/lsprepost"

render_jobs:
  - name: "F6_Bottom_z_section"
    type: section_view
    input: "/data/Tests/Test_001/output/Run_001/d3plot"
    fringe: von_mises
    part_pattern: "PKG*"
    sections:
      - axis: z
        position: center
      - axis: z
        position: 25%
    output:
      format: mp4
      directory: "./renders"
      fps: 30
      resolution: [1920, 1080]

  - name: "E10_Right_Bottom_z_section"
    type: section_view
    input: "/data/Tests/Test_001/output/Run_005/d3plot"
    fringe: von_mises
    part_pattern: "PKG*"
    section:
      axis: z
      position: center
    output:
      format: mp4
      directory: "./renders"
      fps: 30
      resolution: [1920, 1080]
```

### 주요 필드

| 필드 | 설명 |
|------|------|
| `input` | 각 각도(Run)의 d3plot 파일 절대 경로 |
| `type` | `section_view` — 단면 절단 뷰 |
| `fringe` | 컨투어 물리량 |
| `part_pattern` | 표시할 부품 glob 패턴 |
| `sections` / `section` | 단면 정의 (복수/단수). axis + position |
| `output.format` | `mp4`, `png`, `gif` |
| `output.directory` | 출력 디렉토리 (기본 `./renders`) |
| `output.fps` | 프레임 레이트 |
| `output.resolution` | `[width, height]` |

## 실행 방법

### 방법 1: 통합 파이프라인 (권장)

```bash
# 분석 + 리포트 + 렌더링 전체 실행
analyze_and_report /data/Tests/Test_001 --render-config render_config.yaml

# 렌더링만 실행 (분석/리포트 건너뛰기)
analyze_and_report /data/Tests/Test_001 --render-only --render-config render_config.yaml
```

파이프라인은 내부적으로 YAML의 `render_jobs`를 개별 작업으로 분리하여
각 작업마다 `unified_analyzer --config <job.yaml> --render-only`를 순차 실행합니다.

### 방법 2: unified_analyzer 직접 실행

```bash
unified_analyzer --config render_config.yaml --render-only
```

> **주의**: unified_analyzer는 전역 `input.d3plot`을 사용합니다.
> 각 render_job에 다른 d3plot 경로가 필요한 경우 통합 파이프라인을 사용하세요.

## 실행 예시

### 1. 리포트 생성
```bash
python3 -m koo_sphere_report --test-dir /data/Tests/Test_001_Full26_1Step
```

### 2. 리포트에서 YAML 생성
1. 브라우저에서 `report.html` 열기
2. **Render Export** 탭 클릭
3. **Top 5 Stress** 버튼 클릭 (또는 Mollweide에서 직접 선택)
4. 옵션 확인/수정
5. **Download YAML** 클릭

### 3. 렌더링 실행
```bash
cd /data/Tests/Test_001_Full26_1Step
analyze_and_report . --render-only --render-config ~/Downloads/render_config_Test_001.yaml
```

### 4. 결과 확인
```bash
ls renders/
# F6_Bottom_z_section.mp4
# E10_Right_Bottom_z_section.mp4
# ...
```

## LSPrePost 요구사항

렌더링은 내부적으로 **LSPrePost**를 사용합니다.

- LSPrePost 설치 경로: `installed/lsprepost/lsprepost` (또는 YAML의 `lsprepost_path`)
- 라이선스가 활성화되어 있어야 합니다
- Mesa (소프트웨어 렌더링) 환경에서도 동작합니다 (`lspp412_mesa` wrapper 사용)

### 라이선스 문제
```
Maximum number of clients reached
```
이 메시지가 나오면 다른 LSPrePost 인스턴스를 종료하거나 라이선스 서버를 확인하세요.

## 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| YAML에 render_jobs 없음 | 각도를 선택하지 않음 | Mollweide에서 각도를 클릭하거나 Top 5 버튼 사용 |
| `unified_analyzer not found` | 바이너리 미빌드 | `cd build && cmake .. && make -j4 unified_analyzer` |
| 렌더링 결과 없음 | LSPrePost 경로 오류 | YAML의 `lsprepost_path` 확인 |
| `Maximum number of clients` | LSPrePost 라이선스 초과 | 기존 LSPrePost 프로세스 종료 |
| PyYAML 미설치 경고 | `pip install pyyaml` 필요 | 분할 실행 불가 → 단일 모드 폴백 |
| d3plot 경로 오류 | test_dir 이동됨 | YAML 내 input 경로를 수동 수정 |
